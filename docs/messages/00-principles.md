# Принципы дизайна сообщений

> Согласовано в live-ревью 2026-07-22. Источник правды для `bot.view.Texts` / `Keyboards`.

## Контекст

- **Аудитория:** любители ЛАБ (Москва и МО); чаще знают **фамилию**, не ник.
- **MVP в чате:** текст + inline-кнопки + PNG (график); без Mini App.
- **Навигация:** новый экран — новое сообщение (`SendMessage`): поиск → карточка → соперники; внутри экрана соперников (фильтр, пагинация) и H2H-wizard — `EditMessageText`.

## Сквозные правила (согласовано)

### Идентификация игрока

| Правило | Пример |
|---|---|
| **ФИО первым**, ник/id вторичен | `Иванов Иван (Rocket)` |
| В **скобках** — **ссылка** на `https://badminton4u.ru/players/{id}`; текст ссылки = **ник**, иначе **id** | HTML: `<b>ФИО</b> (<a href="…">Rocket</a>)` |
| Нет ФИО | только ссылка: `<b><a …>Rocket</a></b>` или id |
| **Где действует** | любой **текст сообщения** (карточка, H2H — блок игроков, подбор партнёра, заголовок «Соперники») |
| **Исключение 1: строки списка соперников** | в `<pre>` только **ФИО** (или ник/id, если ФИО нет); **без** ника в скобках — ссылки в `<pre>` не кликабельны |
| **Исключение 2: H2H «Фаворит»** | только краткое **ФИО** (или ник/id текстом); **без** повторной ссылки — профили уже выше на экране |
| **Исключение 3: inline-кнопки** | лимит 64 символа — см. [`02-search.md`](02-search.md) |
| В подсказках ввода | «фамилия или ник» (фамилия **первая**) |
| **Кнопки результатов поиска** | `{Фамилия Имя} ({город}) 👤 {r} 👥 {r}`; **без отчества**; усечение: город→👤 — см. [`02-search.md`](02-search.md) §2c |

**Код:** `PlayerDisplayFormat` (plain-подпись для `<pre>`), `Texts.appendPlayerHeader` (HTML), `ProfileLinks.url`.

### Рейтинги

- У игрока **два** рейтинга: **👤** (single, S) и **👥** (double, D). Код S/D на badminton4u — в UI только иконки.
- Не показываем MD/WD/XD на карточке — это разряды **матчей**, не личные рейтинги.
- Строки без значения — **не выводим** (не заполняем `—`).
- Числа рейтинга — **обычный текст** (без `<code>`, без ссылки).

### Счёт и метрики

| Элемент | Формат |
|---|---|
| W–L на экране соперников | `🟢3  🔴1  📊75%` (в `<pre>`, столбцовое выравнивание) |
| Прогноз (H2H) | `Фаворит: Иванов И. (≈62%)` — краткое ФИО **без** повторной ссылки (см. исключение 2) |
| Дельта матча | `Δ +4.5` |
| Дата | `dd.MM.yyyy` |

### Индекс S (сыгранность)

- **Не показываем** на экране соперников и **не показываем строкой** на H2H.
- Используется **только внутри** расчёта прогноза P3 (`FORMULAR.md` §2.3).

### Актуальность данных (дисклеймер)

- **Не** в footer карточки / соперников / H2H.
- **Два места:**
  - `/start` — одна строка: `Данные по Москве и МО · обновление ежедневно`
  - `/help` — развёрнутый блок в конце справки.

### Эмодзи

- **Кнопки меню:** 🔍 🆚 🤝 ℹ️ 🤺 📈 ⬅️ (как в `Keyboards`).
- **Рейтинги:** 👤 👥 (`MessageEmoji.SINGLE` / `DOUBLE`).
- **Форма на карточке:** 👊 (`MessageEmoji.FORM`).
- **Соперники — W/L/%:** 🟢 🔴 📊 (`MessageEmoji.WIN` / `LOSS` / `WIN_RATE`).
- В **теле** карточки — без лишних декоративных эмодзи (📍 и т.п.).

### HTML Telegram

- `<b>` — ФИО в заголовках секций.
- `<a href="…">` — ник/id **в скобках после ФИО** (кроме строк списка соперников в `<pre>`).
- `<pre>` — таблица соперников: `{n}. {ФИО}` + столбец метрик; pad по `PlayerDisplayFormat.rivalsRowLabel`.
- `<code>` — по необходимости на других экранах; **не** для рейтингов на карточке.

## Навигация

| Экран | Кнопка «назад» |
|---|---|
| Соперники | `⬅️ К карточке` |
| H2H (вход с карточки) | `⬅️ К карточке` + `Изменить соперника` |
| H2H (вход `/h2h`) | `⬅️ В меню` + `Изменить соперника` |
| График | `⬅️ К карточке` |

---

## Результат турнира на карточке

1. Призовое место **1–3** → `1-е место`, `2-е место`, `3-е место`.
2. Иначе — стадия вылета из **последнего матча** игрока на турнире: `вылет в 1/4`, `не вышел из группы`, …
3. Нет матчей → `участие без матчей`.

*(Требует доработки бэкенда: `match.stage` для последнего матча на турнире.)*

## H2H — scope (этап 7 DoD)

- **Игрок vs игрок** в выбранной **дисциплине** (шаг 3 wizard: D / MD / WD / XD).
- **Тип пары** на экране — `PairCompositionService`; предупреждение при несоответствии разряду.
- **Пара vs пара** (4 игрока) — отдельный сценарий этапа 7.
- «Последние встречи» = матчи между этой парой игроков в выбранном разрезе дисциплины.

---

## Соперники — фильтр «Все»

- Чип **«Все»** + MD / WD / XD / D.
- **«Все»:** W–L **суммируется** по всем разрядам на соперника; одна строка на человека.
- **Город** в списке соперников **не показываем**.
- **ФИО соперника** в списке — **только ФИО** (нет ФИО → ник или id текстом), без ссылки и без ника в скобках.
