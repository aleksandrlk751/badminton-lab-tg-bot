## 2c. Результаты поиска

**Заголовок** (склонение `найден` / `найдено` / `игрок` / `игрока` / `игроков`):

```
Нашёл 3 игроков, подходящих под критерии поиска
Выберите из предложенных вариантов
```

| n | Заголовок |
|---|---|
| 1 | `Нашёл 1 игрока, подходящего под критерии поиска` |
| 2–4 | `Нашёл N игрока, подходящих под критерии поиска` |
| 5+ | `Нашёл N игроков, подходящих под критерии поиска` |

**Кнопки** (≤64 символа — лимит Telegram; при переполнении см. ниже):

```
Иванов Иван (Москва) 🆂 320 🅳 535
```

**Шаблон:**

```
{ФИО} ({город}) 🆂 {rating_s} 🅳 {rating_d}
```

| Часть | Правило |
|---|---|
| ФИО | обязательно; если пусто → **ник**; если и ника нет → `Игрок {id}` |
| Город | в скобках; если пусто — скобки **не выводим** |
| 🆂 / 🅳 | как на карточке: **только если рейтинг есть** |
| Ник | **не** в кнопке (акцент на фамилию) |

**Примеры:**

```
Иванов Иван (Москва) 🆂 320 🅳 535
Петров Пётр (Химки) 🅳 412
Сидорова Анна 🆂 280 🅳 390          ← нет города — скобки не выводим
Rocket 🅳 500                       ← нет ФИО в БД — показываем ник
```

**Скобки `(...)`** — **только город**. Id в кнопке поиска **не используем** (id — запасной текст ссылки на **карточке**, когда нет ника).

**Усечение** (лимит Telegram — **64 символа** на подпись кнопки).

Полный формат всегда собирается как `{ФИО} ({город}) 🆂 {rating_s} 🅳 {rating_d}`.
Если не влезает — **по порядку**:

| Шаг | Действие | Было → стало |
|---|---|---|
| 1 | Убрать **отчество полностью** | `Иванов Иван Олегович` → `Иванов Иван` |
| 2 | Убрать **город** (и скобки) | `(Москва)` → — |
| 3 | Убрать **🆂 рейтинг** | `🆂 320` → —; **🅳 остаётся** |

**Пример цепочки:**

```
Иванов Иван Олегович (Москва) 🆂 320 🅳 535   ← полный
Иванов Иван (Москва) 🆂 320 🅳 535            ← шаг 1
Иванов Иван 🆂 320 🅳 535                     ← шаг 2
Иванов Иван 🅳 535                            ← шаг 3
```

Если после шага 3 всё ещё >64 — обрезка «…» в конце (крайний случай).
*(Реализация: `PlayerSearchResult` + рейтинги S/D в `PlayerSearchService`, `Keyboards.resultLabel()`.)*

## Поиск (техника)

Нечёткий поиск `PlayerSearchService` + `pg_trgm`: фамилия, имя, ник, от 3 символов, до 10 результатов.
H2H-wizard использует **тот же** поиск + список кнопок при нескольких совпадениях.
