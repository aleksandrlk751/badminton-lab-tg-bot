# AGENTS.md

Инструкции для AI-агентов, работающих над этим репозиторием.

## О проекте

Telegram-бот для любителей бадминтона (ЛАБ): аналитика игроков/турниров, сыгранность соперников,
подбор партнёров, граф связанности, лайв-помощник. Подробности, все принятые решения, формулы
и открытые вопросы — в **[`docs/BRIEF.md`](docs/BRIEF.md)**. План этапов — **[`docs/PLAN.md`](docs/PLAN.md)**.
Формулы и константы (дефолты, локализация в коде) — **[`docs/FORMULAR.md`](docs/FORMULAR.md)**.
Всегда сверяйся с брифом перед работой и обновляй его при принятии новых решений.

**Статус:** реализация. **Этапы 0–2 завершены** — spike: [`docs/spike-parser.md`](docs/spike-parser.md). 
Модули: **`core`** (JPA, Flyway), **`worker`** (парсер + слепок r77), **`bot`** (Telegram). Локальный запуск: **`dev.ps1`**. Следующий — этап 3 (метрики: S, форма, прогноз P3).

## Как общаться с пользователем

- Отвечай **на русском**.
- Пользователь — разработчик ПО на **Java**, объясняй на уровне middle-специалиста.
- Если вопрос пользователя выглядит как запрос на детали/уточнения, а явного указания править код нет —
  сначала уточни необходимость правок, потом действуй.

## Ключевые правила проекта

1. **Не выдумывай формулы и алгоритмы.** Если реализация требует неопределённой формулы/алгоритма —
   не действуй наугад: предложи пользователю варианты и уточни выбор.
2. **Официальные формулы ЛАБ зеркалим как есть** (не заменяем своими):
 - дельта одиночки: `(100 - (R_поб - R_проигр)) / 10`, при разрыве ≥100 → 0, пол = 1;
 - рейтинг пары: `(A + B) / 2`.
 Собственные метрики (индекс сыгранности, форма, прогноз) — надстройка сверху.
 Все формулы, константы и их локализация — в [`docs/FORMULAR.md`](docs/FORMULAR.md); дефолты метрик
 живут в `application-core.yml` (`badminton-lab.metrics.*`) / `MetricsProperties`.
3. **Сторонние библиотеки** (на любом языке) — перед использованием изучай их актуальную
   документацию через context7.
4. **Парсинг источников — только вежливо:** rate-limit, кеширование, честный User-Agent с контактом.
   Не создавай агрессивную нагрузку на `badminton4u.ru` / `badminton77.ru`.

## Источник данных (справка)

- Основной: `badminton4u.ru` — серверный HTML, стабильные числовые ID.
  Ключевые страницы: `tournaments/`, `tournaments/{id}`, `players/`, `players/{id}`,
  `games/?user1ID=X&user2ID=Y`, **`gamesd/?tourID={id}`** (pair-vs-pair; источник матчей и соперников),
  `content/2`, `content/10`.
- **Spike (этап 0):** pair-vs-pair — SSR на `gamesd/?tourID=`; соперники — **только** из матчей турниров
  (не `/rivals`); регистрация пар — SSR в `#tour-reg-list1` или AJAX `POST /?ajax`.
  Подробности — [`docs/spike-parser.md`](docs/spike-parser.md).
- Детальная сетка турнира и счёт по очкам в партии — только картинками (`res-N.png`); в тексте
  доступен счёт по сетам.
- Старт по региону Москва и МО (`cities[]=r77`).
